Elevation

Borders first — shadows clarify layer, not structure
xs --shadow-xs Resting state — block rows, avatar, small surfaces. Nearly invisible on light backgrounds.
sm --shadow-sm Raised elements — selected block row, phone frame in playground when hovered.
md --shadow-md Dropdowns, popovers, floating toolbars. Floats clearly above the base surface.
lg --shadow-lg Modals, dialogs, the phone preview frame. Highest level — use sparingly.

Borders over shadows. All four cards above also carry a 1px border (--border). That border is the primary structure signal — the shadow adds depth. In dark mode, shadows deepen significantly (opacity 0.30-0.55) while borders lighten, maintaining the same visual hierarchy without needing extra shadow layers.

Motion

Transform + opacity only. Respects prefers-reduced-motion.

Hover

Background transitions from --surface-2 to --accent-subtle using var(--dur-base) + var(--ease-out) (180ms expo-out). This is the same transition used on hovered block rows in the content panel — fast enough to feel instant, long enough to feel intentional.

Rules. Animate only transform and opacity for performance — avoid animating width, height, or layout properties. Live preview debounces text inputs ~120ms to avoid flickering on fast typists; toggles and select changes apply instantly. Block drag-and-reorder uses a spring (no CSS token — handled by the dnd library). All durations collapse to 0.01ms when prefers-reduced-motion: reduce is set.
